    Basebox Auto Mode

    2019-12-02

    Basebox 基盒內的軟件執行上可分為 自動執行模式 及 非自動執行模式。

    自動執行模式 auto mode

    情景

    一般正式使用情景下 Basebox 軟件執行上是在 自動執行模式。

    在此模式下，每當 Basebox 接上電源或重啟時，它會自動做下列事項

    • 載入 MOS 1
    • 載入 Base Libraries
    • 載入 設定檔個
    • 啟動 WiFi, LED 等基礎功能
    • 啟動 收集機械訊號功能
    • 執行 WiFi 連線
    • 與服務器同步時鐘
    • 登入 指定接口 (MI)
    • 重複 收集發送數據

    非自動執行模式 stop-auto mode

    情景

    當有需要對 Basebox 軟件或個別配置作修改時，可把它設為 非自動執行模式。

    在此模式下，每當接上電源或重啟時，Basebox 軟件只會載入 MOS，其他程式庫，設定及腳本均不載入或執行。

    設為此模式的好處時，在 Basebox 安裝修改程式或設定時，不會被其他 自動執行模式 會運行的程序影響。

    Tips

    當 Basebox 重啟時，如果你在 toolz 看見啟動內容出現以下句子，表示它啟動在 非自動執行模式:

    lua: cannot open init.lua

    切換執行模式

    你可以用 toolz 的 預設指令 auto 及 stop-auto 來切換執行模式

    切換到 自動執行模式

    tool usb>cmd auto

    切換到 非自動執行模式

    tool usb>cmd stop-auto

    Note

    如果通過 預設指令 由一個模式切換另一個模式，Basebox 會重啟。

    檢查執行模式

    你可以用 toolz 的 預設指令 check-mode 來檢查 Basebox 當前的執行模式

    tool usb>cmd check-mode
    如果出現 mode = auto 句字，表示當前是 自動執行模式。

    如果出現 mode = stop-auto 句字，表示當前是 非自動執行模式。

    1. MOS - 可以簡單理解為 Basebox 的操作系統 (operating system)，又或類似你電腦上的 Windows。 ↩
